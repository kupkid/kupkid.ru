golang thoughts

мысли о golang

2025-01-28

two months writing go. mixed feelings, but leaning towards it being a good tool for specific tasks.

два месяца пишу на go. впечатления смешанные, но склоняюсь к тому, что это хороший инструмент для определенных задач.

pros

плюсы

  • single binary compilation — deploy is just scp
  • компиляция в один бинарник — деплой это просто scp
  • goroutines — concurrency actually simple
  • горутины — concurrency действительно простой
  • standard library — almost everything in the box
  • стандартная библиотека — почти всё есть из коробки
  • go fmt — no holy wars about style
  • go fmt — никаких holy wars про стиль

cons

минусы

  • error handling — if err != nil everywhere
  • обработка ошибок — if err != nil повсюду
  • no generics (though 1.18+ has them now)
  • отсутствие generics (хотя уже есть в 1.18+)
  • gopath and modules — still confusing
  • путь gopath и модули — до сих пор путаюсь
  • nil — million nil checks
  • nil — миллион проверок на nil

conclusion

вывод

go is c for the 21st century. not beautiful, not convenient, but damn efficient. for backends and microservices — perfect. for everything else — better pick something else.

go — это c для 21 века. не красивый, не удобный, но чертовски эффективный. для бэкендов и микросервисов — идеален. для всего остального — лучше взять что-то другое.

staying on python for bots and scripts, but starting new projects in go. a compromise i can live with.

пока остаюсь на python для ботов и скриптов, но новые проекты начинаю на go. компромисс, с которым можно жить.
